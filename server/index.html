Lottie Env

A reinforcement learning environment for training agents to generate Lottie animations.

The Task

An RL agent must learn to produce valid Lottie JSON that, when rendered, visually matches a set of reference frames. On each episode the environment picks a random animation task and presents three keyframes — start, middle, and end — as the observation. The agent then submits its Lottie JSON and receives a similarity reward.

Episode Loop

POST /reset
→
Observation: 3 reference frames
→
POST /step with Lottie JSON
→
Reward + submitted frames

Each episode is single-step: the agent gets one chance to submit a Lottie animation and receives a reward. A new reset() picks a fresh task.

Reward

ConditionReward
Valid Lottie JSON → rendered frames compared to reference 0.0 – 1.0 MSE-based similarity
Invalid JSON or schema validation fails −1.0

The reward is computed from the mean Mean Squared Error (MSE) across all three frame pairs (start, middle, end). A perfect match (MSE = 0) scores 1.0; random noise will be close to 0.0. The reward is transformed as: 1.0 - min(MSE / 65025, 1.0) where 65025 is the maximum possible MSE for RGB images. Submitted frames are resized to match reference dimensions before comparison.

How It Works

  1. Schema validation — the submitted JSON is validated against the official Lottie JSON Schema (via lottie-specs).
  2. Frame extraction — valid animations are rendered with rlottie-python to extract 3 keyframes (first, middle, last).
  3. MSE comparison — each submitted frame is compared pixel-by-pixel to the corresponding reference frame using mean squared error.
  4. Reward — the mean MSE across all 3 pairs is transformed to a similarity score in [0, 1].
